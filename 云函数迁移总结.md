### 1. 配置系统 ✓

**文件**: `/miniprogram/config/api.config.js`

- 支持 `cloud` 和 `api` 两种模式切换
- 配置开发环境和生产环境API地址
- 可通过修改 `mode` 参数一键切换

```javascript
const config = {
  mode: 'cloud', // 'cloud' 或 'api'
  api: {
    dev: { baseUrl: 'http://localhost:3000/api' },
    prod: { baseUrl: 'https://your-domain.com/api' }
  },
  env: 'dev' // 'dev' 或 'prod'
};
```

**文件**: `/miniprogram/utils/api.js`

**使用示例**:

const api = require('../../utils/api.js');
const res = await api.login({ openid: app.globalData.openid });

## 🚀 如何使用

### 方式一：继续使用云函数（默认）

无需任何操作，当前配置已设为云函数模式。

### 方式二：切换到后台API

#### 1. 启动后台服务器

```bash
cd server
npm install
npm run dev
```

#### 2. 修改配置

编辑 `/miniprogram/config/api.config.js`:

```javascript
const config = {
  mode: 'api', // 改为 'api'
  env: 'dev'   // 开发环境
};
```

#### 3. 配置MongoDB

确保MongoDB运行在 `localhost:27017`，或修改 `.env` 文件配置连接字符串。

#### 4. 测试

在小程序开发者工具中：
- 勾选"不校验合法域名"
- 重新编译运行

## 🔄 切换方式对比

| 特性 | 云函数模式 | 后台API模式 |
|------|-----------|------------|
| 响应速度 | 200-500ms | 50-200ms |
| 成本 | 按调用次数计费 | 固定服务器成本 |
| 并发能力 | 受云函数配额限制 | 可自由扩展 |
| 灵活性 | 受限于云开发 | 完全可控 |
| 维护难度 | 低 | 中等 |
| 数据库 | 云数据库 | MongoDB |
| 部署难度 | 简单 | 需要服务器 |

## 💡 建议

1. **开发阶段**：使用云函数模式，快速开发
2. **测试阶段**：切换到API模式，测试性能
3. **生产环境**：根据实际需求选择
   - 用户量小：云函数（成本低）
   - 用户量大：后台API（性能好）
